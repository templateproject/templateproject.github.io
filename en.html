Project Summary

Information Retrieval Group

In classical/traditional information retrieval, a single retrieval strategy is applied uniformly to all information needs (i.e., queries). However, every single retrieval strategy performs well on some queries while performing the others poorly - robustness in retrieval performance problem. On the other hand, a query that a retrieval strategy performs poorly is not necessarily performed poorly by the other retrieval strategies: for any given query, it is possible to find a retrieval strategy with satisfactory performance among existing technologies. The notion of selective information retrieval is based on this (empirical) fact. Compared to classical/traditional information retrieval, selective information retrieval promises two or three times better effectiveness on the average as well as a satisfactory performance for every query. It is unlikely for any other existing retrieval technology to effort that much improvement. Unfortunately, this potential has not been fully exploited yet. This line of information retrieval research is relatively new and the vast amount of the works is, for now, of academic type due to the lack of mature practical technologies. Nonetheless, after the initial works on the issue of robustness in retrieval performance during TREC 2003-2005 Robust Track studies, the fact that the TREC 2013 Web track addresses the same issue with a sub-task called the “Risk-Sensitive Evaluation” suggests that, sooner or later, it will eventually find its way to the market. The aim of this project is

  • to develop a statistically motivated selective information retrieval system and
  • to select the best retrieval strategy with high accuracy for any given query.

The novelty of the proposed work can be summarized as follows:

  • taking into account the relationships among the components of information retrieval (pre-process, matching and post-process) for selective information retrieval,
  • being able to configure the retrieval strategy, on the fly, that best fits to any query given,
  • exploiting a relatively large set of candidate retrieval technologies for selective retrieval,
  • use of a new set of query features, to be grounded upon the theory of statistics/probability, in order to select best index term weighting model in particular and to select best retrieval strategy in general.

Since the work proposed in this project is novel, it is expected that it leads a commercial product, in case that the project is concluded with success. In addition, if the knowledge to be generated by the project is transferred to a local organization or company, it contributes to the organization’s competition power in the world-wide market. For the main product of this project, which is the core part of any information retrieval system, it is also highly likely to be used by e-commerce applications.

Keywords: Selective Information Retrieval, Information Retrieval, Robustness in Retrieval Performance
